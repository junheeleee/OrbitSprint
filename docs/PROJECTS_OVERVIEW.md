## 추천 기록 방식

현재 단계에서는 Notion이나 Slack보다 **GitHub + repo 내부 문서**를 기준으로 두는 것이 가장 좋습니다.

- GitHub: 코드, 커밋, 이슈, 릴리즈, 버그 기록의 기준
- Repo `docs/`: 요구사항, 의사결정, 작업 로그의 기준
- Notion: 투자자/기획서/스토리보드처럼 보기 좋게 정리할 때 보조 사용
- Slack: 혼자 개발하거나 소규모 협업 전에는 필수 아님

권장 구조:

```text
docs/
├── PROJECTS_OVERVIEW.md  # 두 프로젝트 전체 현황
├── WORK_LOG.md           # 날짜별 해결 과정
├── DECISIONS.md          # 중요한 기획/기술 결정
└── ROADMAP.md            # 다음 작업 우선순위
```

## Project 1: Lumen Run

기존 이름은 Orbit Sprint였고, IP/비주얼/게임성 방향을 정리하면서 **Lumen Run**으로 통일했습니다.

### 핵심 방향

- iOS App Store 출시 목표
- 원버튼 궤도 액션 게임
- SF풍 고속 아케이드 감각
- 짧은 세션, 점수 경쟁, 피버 모드 중심
- 한국어/영어 지원

### 주요 요구사항

- 번들 ID, Xcode signing, 실제 iPhone 테스트 가능 상태
- 사운드 토글과 실제 음량/효과음 동작 정리
- 3단계 궤도 시스템
- `1 -> 2 -> 3 -> 2 -> 1` 방식의 궤도 이동
- 같은 레이어 장애물만 충돌
- 플레이어 진행 방향에 즉시 생성되는 불공정 장애물 방지
- 충돌 후 같은 장애물에 연속 사망하지 않도록 처리
- 로딩 화면과 스타트 화면
- 통일된 IP, 제목, 음악, 효과음, 피버 모드
- 피버 중 무적, 고속 이동, 충돌 대상 점수화

### 현재 상태

- GitHub 원격: `git@github.com:junheeleee/LumenRun.git`
- 프로젝트명/폴더명: `LumenRun`
- 최근 커밋 기준으로 빌드/푸시 진행됨
- 주요 문제였던 충돌 판정, 궤도 버벅임, 배경음 템포는 한 차례 정리됨

## Project 2: Gangnam Dream

`Gangnam Dream`은 Godot 기반 한국 현대생활 로그라이크 텍스트 RPG입니다.

### 핵심 방향

- 한국 현대 사회 생존 시뮬레이터
- Football Manager식 대시보드 UI
- 비주얼 노벨식 이벤트 선택
- 금융/취업/관계/스트레스 중심
- 어두운 사회 풍자와 반복 플레이성

### 주요 요구사항

- Godot 4.6 기준
- JSON 기반 데이터 드리븐 이벤트 시스템
- 이벤트 조건, 희귀도, 가중치, 쿨다운, 히든 이벤트, 연쇄 이벤트
- 월별 뉴스 생성과 시장 영향
- 투자 시스템: 변동성, 버블, 폭락, 공포/탐욕, 모멘텀
- 관계 시스템: 가족, 친구, 연애, 직장, 멘토, 사업 파트너
- 다중 저장 슬롯과 자동저장
- 메타 진행도, 해금 특성, 업적, 런 히스토리
- 한국어 현대생활 이벤트 대량 생성

### 현재 상태

- 위치: `gangnam_dream/`
- Godot 프로젝트가 기존 Claude 임시 폴더에서 Game 폴더 아래로 이동됨
- 현재는 LumenRun repo 안에 함께 추적되어 있음
- 별도 GitHub repo 생성은 아직 필요
- 현재 GitHub 자동 생성은 `gh` CLI 없음, SSH publickey 권한 없음으로 막힘
